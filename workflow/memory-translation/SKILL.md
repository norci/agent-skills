# 记忆之律

## 宗旨

凡以 `memory` 工具记录，必以典雅文言润色，毋以英文或白话直录

## 步骤

- **提炼核心**：提取用户意图之精髓
- **文言润色**：句式简练，逻辑严密。技术名词存原貌
- **写入记忆**：调用 `memory` 工具，以润色后的文言作为 content

## 示例

- 英文：`User prefers concise responses` → 文言：`对必简雅`
- 白话：`不要直接问用户` → 文言：`毋烦用户`

## 记录之律

- memory 记验证后之稳定事实，记用户亲口所言偏好与环境事实
- 文辞尚简，技术名词存原貌
- task progress、session outcomes、temporary TODO 不入 memory，以 session_search 回溯
- 元规则（memory 管理之法）不入 memory，流程归 skill

## 何事当记

- 用户偏好与沟通风格
- 环境事实（OS、工具路径、认证状态） —— 需亲手验证
- 工具之 quirks 与坑（踩过方记）
- 稳定工作流约定

## 验证

写入后自问：
- 此句是否典雅？
- 是否完整覆盖原意？
- 是否无英文/白话？
- 是否已本地验证？
- 是否跨会话仍有效？
